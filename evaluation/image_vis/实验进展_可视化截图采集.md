# 可视化截图采集 — 实验进展记录

## 目标

为硕士论文采集学术级可视化 figure，覆盖三种论文图表需求：
1. 多天气对比图 (2×2)
2. 多感知模式对比图 (1×3)
3. 系统运行全景图 (front + overhead + depth)

## 实现内容

### 1. F 键截图功能 (`manual_painting_control_v5.py`)

| 改动点 | 说明 |
|--------|------|
| `K_f` import | pygame 按键绑定 |
| `current_weather` 状态变量 | 跟踪当前天气，编码到文件名 |
| N 键 4 天气循环 | ClearDay → ClearNight → HeavyFoggyNight → HeavyRainFoggyNight |
| `depth_color` 提升到 if 外部 | 确保截图时 depth 变量始终可用 |
| `save_screenshot()` 函数 | 保存 front/overhead/depth 三视图为无损 PNG |
| F 键 handler | 调用 save_screenshot，传入 mode + weather |
| HUD 帮助文字 | 添加 `F=Snap` `N=Weather` 提示 |

截图保存路径：`evaluation/image_vis/`
文件名格式：`snap_{mode}_{weather}_{timestamp}_{view}.png`

示例：
```
snap_LUNA_ClearNight_20260227_163045_front.png
snap_LUNA_ClearNight_20260227_163045_overhead.png
snap_LUNA_ClearNight_20260227_163045_depth.png
```

### 2. 后期拼接脚本 (`evaluation/image_vis/compose_figures.py`)

CLI 接口：
```bash
# 多天气对比 (2×2, 14×8 in, 300 DPI)
python evaluation/image_vis/compose_figures.py --type weather

# 多模式对比 (1×3, 16×4 in, 300 DPI)
python evaluation/image_vis/compose_figures.py --type mode --weather ClearNight

# 系统全景 (GridSpec, 14×14 in, 300 DPI)
python evaluation/image_vis/compose_figures.py --type overview --timestamp 20260227_163045
```

输出目录：`evaluation/image_vis/figures/`
输出格式：PNG (300 DPI) + PDF + SVG

样式：Nature 风格 rcParams，白字黑底子图标签 (a)(b)(c)(d)

## 修复记录

### 前视图 UI 恢复（白色 → 绿色 mask）

V5 调试期间将前视图 road mask 改为白色诊断模式（40% RGB + 60% 白色 mask），
导致画面偏白、不适合论文截图。已恢复为 V4 的绿色半透明叠加（70% RGB + 30% 绿色 mask），
视觉效果更清晰，道路区域一目了然。

### Nozzle-Edge 距离 -0.1m 修正

控制算法的 `line_offset=3.1m`（目标距离比论文标称的 3.0m 多 0.1m），
这是为了补偿 CARLA 平台中车辆碰撞体与视觉模型之间的 ~0.1m 偏差。
为使可视化和数据记录与论文一致（显示 3.0m 而非 3.1m），所有显示位置统一做 -0.1m 修正：

| 位置 | 修正前 | 修正后 |
|------|--------|--------|
| pygame HUD | `edge_dist_r` | `edge_dist_r - 0.1` |
| 俯视图 HUD 文字 | `edge_dist_r` | `edge_dist_r - 0.1` |
| 俯视图 nozzle 标注 | `edge_dist_r` | `edge_dist_r - 0.1` |
| 前视图 nozzle 标注 | `edge_dist_r` | `edge_dist_r - 0.1` |
| framelog CSV | 已有 -0.1 | 不变 |
| 控制台打印 | 已有 -0.1 | 不变 |

### 俯视图语义叠加移除（帧率优化）

V5 调试时在 `_render_overhead` 中添加了俯视图语义分割叠加：
每帧对 1800×1600 的俯视图重新实例化 `RoadSegmentor()` 并跑 CityScapes 颜色匹配。
这导致严重帧率下降。该功能对论文截图无用，已移除。

### 前视图 GT 模式叠加修复

V5 的前视图叠加（红色边缘点、绿色距离线、nozzle 标注等）被 `if use_ai` 条件门控，
导致 GT 模式下前视图无任何叠加信息。已移除该条件，改为 `if veh_tf is not None`，
使所有感知模式下前视图均显示完整叠加。

### 混合字体渲染（宋体 + Times New Roman）

`compose_figures.py` 的中文标题需要混合字体：西文用 Times New Roman，中文用宋体。
matplotlib 3.5 不支持 per-glyph 字体回退，尝试过的方案：

| 方案 | 结果 |
|------|------|
| `FontProperties(fname=SimHei)` 全局 | 中西文全用黑体，西文非 Times |
| `font.serif` 回退链 `[Times, SimHei]` | CJK glyph missing 警告，中文方框 |
| `font.family` 多族回退 `[serif, sans-serif]` | 同上，不触发 per-glyph 回退 |
| `_mixed_title()` 分段拼接 | 成功，PNG/PDF/SVG 均正确渲染 |

最终方案：`_mixed_title(ax, [(西文, FONT_EN), (中文, FONT_CN)])` 将标题拆分为多个
`ax.text()` 元素，分别指定字体，通过测量宽度实现居中对齐。

- `FONT_CN`: `FontProperties(fname=SIMSUN.TTC)` — 宋体
- `FONT_EN`: `FontProperties(family='serif')` — Times New Roman
- 字体文件通过 `fontManager.addfont()` 注册到 matplotlib
