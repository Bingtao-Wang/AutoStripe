# AutoStripe V4.2 — 论文级数据记录系统

**版本**：V4.2（数据记录增强）
**日期**：2026-02-11
**基于**：V4.2 PD 控制 + 评估管线 + 虚线模式

---

## 一、版本定位

V4.2 在 PD 控制器、轨迹评估和虚线模式的基础上，新增**论文级数据记录系统**，
为顶会论文（ICRA / IROS / T-RO / RA-L）提供完整的实验数据支撑。

| 能力 | V4.2 之前 | V4.2 数据记录增强 |
|------|-----------|-------------------|
| 评估数据 | E 键快照（summary + 3 列 detail CSV） | E 键录制（summary + 8 列 detail + 31 列 framelog） |
| 推理耗时 | 无 | 逐帧 ms 级精度（CUDA synchronize） |
| 感知精度 | 无 | mask IoU + edge deviation（AI vs GT，逐帧） |
| 时序分析 | 无 | 8 子图时序图（距离/偏移/转向/速度/误差/推理/IoU/边缘偏差） |
| 空间分析 | 无 | 7 张地图可视化（轨迹/覆盖/状态/偏差/分段/控制器/速度曲率） |
| 曲率分析 | 仅全局曲率方差 | 逐点曲率 + 曲率-偏差散点图 |
| GT 对应 | 仅最近距离 | 最近 GT 坐标 + 沿轨迹距离 + 范围标记 |
| 输出管理 | evaluation/ 根目录混杂 | run_\<timestamp\>/ 子目录隔离 |

## 四、逐帧记录器（FrameLogger）

### 4.1 CSV 格式（31 列）

```
timestamp, frame, dt,
veh_x, veh_y, veh_yaw, speed,
nozzle_x, nozzle_y, nozzle_edge_dist, poly_edge_dist,
driving_offset, steer_filter, steer_cmd, throttle_cmd, brake_cmd,
lateral_error, paint_state, painting_enabled, dash_phase,
perception_mode, ai_edge_pts, gt_edge_pts, road_mask_ratio,
poly_coeff_a, poly_coeff_b, poly_coeff_c,
inference_time_ms,
mask_iou, edge_dev_mean_px, edge_dev_median_px, edge_dev_max_px
```

### 4.2 字段说明

| 分类 | 字段 | 类型 | 说明 |
|------|------|------|------|
| **时间** | timestamp | float | Unix 时间戳 |
| | frame | int | 帧序号 |
| | dt | float | 帧间隔（秒） |
| **车辆** | veh_x, veh_y | float | 车辆世界坐标 |
| | veh_yaw | float | 车辆航向角（度） |
| | speed | float | 车速（m/s） |
| **喷嘴** | nozzle_x, nozzle_y | float | 喷嘴世界坐标 |
| | nozzle_edge_dist | float | 喷嘴到路缘中值距离（m） |
| | poly_edge_dist | float | 多项式外推距离（m），无效时 -1 |
| **控制** | driving_offset | float | 当前驾驶偏移量（m） |
| | steer_filter | float | 当前转向滤波系数 |
| | steer_cmd | float | 转向指令 |
| | throttle_cmd | float | 油门指令 |
| | brake_cmd | float | 刹车指令 |
| | lateral_error | float | 横向误差 = (poly_dist - nozzle_arm) - 3.0 |
| **状态** | paint_state | str | CONVERGING / STABILIZED / PAINTING |
| | painting_enabled | int | 0 或 1 |
| | dash_phase | int | 1=画线 / 0=间隔 / -1=非虚线 |
| **感知** | perception_mode | str | AI 或 GT |
| | ai_edge_pts | int | AI 检测到的路缘点数 |
| | gt_edge_pts | int | GT 参考路缘点数 |
| | road_mask_ratio | float | 道路像素占比 |
| **多项式** | poly_coeff_a/b/c | float | 二次拟合系数 lat = a·lon² + b·lon + c |
| **性能** | inference_time_ms | float | VLLiNet 推理耗时（ms），skip 帧为 -1 |
| **感知精度** | mask_iou | float | AI road mask vs GT road mask 的 IoU (0~1) |
| | edge_dev_mean_px | float | AI 右路缘 vs GT 右路缘逐行像素偏差均值 |
| | edge_dev_median_px | float | 同上，中位数 |
| | edge_dev_max_px | float | 同上，最大值 |

### 4.3 输出路径

```
evaluation/run_20260211_041712/framelog_20260211_041712.csv
```

每次 E 键录制自动创建 `run_<timestamp>/` 子目录，所有输出文件隔离存放。

### 4.4 API

```python
from evaluation.frame_logger import FrameLogger

logger = FrameLogger()
logger.start()                    # 创建 CSV，写表头
logger.log_frame({'frame': 1, 'speed': 2.3, ...})  # 写一行
logger.stop()                     # flush + 关闭
print(logger.active)              # bool: 是否正在录制
```

## 五、增强评估 Detail CSV（8 列）

### 5.1 新格式

```
paint_x, paint_y, nearest_gt_dist,
gt_nearest_x, gt_nearest_y,
along_track_dist, local_curvature, in_range
```

### 5.2 新增列说明

| 列 | 类型 | 说明 |
|----|------|------|
| gt_nearest_x | float | 最近 GT 点 X 坐标 |
| gt_nearest_y | float | 最近 GT 点 Y 坐标 |
| along_track_dist | float | 从第一个 paint 点的累积沿轨迹距离（m） |
| local_curvature | float | Menger 曲率（三点外接圆，1/m） |
| in_range | int | 1 = nearest_gt_dist < 5.0m，0 = 超出 GT 覆盖 |

### 5.3 向后兼容

`visualize_eval.py` 的 `load_detail()` 自动检测列数：
- 3 列（旧格式）→ 返回 `(xs, ys, dists)` tuple
- 8 列（新格式）→ 返回 `dict`，key 为列名

## 六、感知推理计时

### 6.1 实现

```python
# road_segmentor_ai.py
@torch.no_grad()
def _infer(self, rgb_tensor, depth_tensor, orig_h, orig_w):
    t0 = time.time()
    # ... model forward ...
    if self.device.type == 'cuda':
        torch.cuda.synchronize()   # 等待 GPU 完成
    self.last_inference_ms = (time.time() - t0) * 1000.0
```

### 6.2 数据流

```
RoadSegmentorAI.last_inference_ms
    → PerceptionPipeline.last_inference_ms（AI 模式转发，GT 模式 = 0）
        → frame_logger.log_frame({'inference_time_ms': ...})
```

### 6.3 注意事项

- `torch.cuda.synchronize()` 必须在计时结束前调用，否则 GPU 异步执行导致计时偏小
- Skip-frame 帧不运行推理，framelog 中记录 -1
- 预期值：~20-40ms（RTX 3090），~50-80ms（GTX 1080 Ti）

## 九、主循环集成流程

### 9.1 E 键操作流程

```
第一次按 E：
    eval_recording = True
    frame_logger.start()        → 创建 framelog CSV
    eval_trail_start_idx = len(paint_trail)

每帧（eval_recording == True）：
    frame_logger.log_frame({    → 写 27 列数据
        timestamp, frame, dt,
        veh_x, veh_y, ...
        inference_time_ms
    })

第二次按 E：
    eval_recording = False
    frame_logger.stop()         → 关闭 framelog CSV
    segment = paint_trail[start_idx:]
    evaluator.run_evaluation(segment, vehicle_location)
        → summary CSV + 8 列 detail CSV + 控制台打印
```

## 十一、论文数据使用指南

### 11.1 控制器性能分析

从 `framelog` 提取：
- **收敛曲线**：`nozzle_edge_dist` vs `frame`，标注 paint_state 转换点
- **PD 响应**：`driving_offset` + `lateral_error` 叠加图
- **稳态精度**：PAINTING 状态下 `nozzle_edge_dist` 的均值/标准差

### 11.2 感知性能分析

从 `framelog` 提取：
- **推理延迟**：`inference_time_ms` 的均值/P95/P99
- **感知稳定性**：`ai_edge_pts` 和 `road_mask_ratio` 随时间变化
- **Skip-frame 影响**：对比推理帧和缓存帧的 `nozzle_edge_dist` 差异

### 11.3 轨迹精度分析

从 `detail` 提取：
- **整体精度**：mean/median/std/max deviation（summary CSV）
- **空间分布**：XY 轨迹图，颜色编码偏差（viz.png）
- **曲率相关性**：curvature vs deviation 散点图（curv_dev.png）
- **沿轨迹变化**：deviation vs along_track_dist 曲线

### 11.4 对比实验

多次实验的 summary CSV 可用 `plot_multi_session_comparison()` 自动生成对比柱状图。
建议对比维度：
- AI vs GT 感知模式
- 不同出生点（直道 vs 弯道）
- 实线 vs 虚线模式
